실시간 시스템 제어 및 컴퓨터 비전 분석 프로젝트

고성능 제어 시스템 설계를 위한 컴퓨터 비전 심화 분석 연구 보고서

본 프로젝트는 실시간 제어 및 인공지능 로직 학습만을 목적으로 합니다. (순수 연구용)

1. 프로젝트 개요 및 동기 (왜 만들었나?)

이 연구는 **컴퓨터 비전(CV)** 기술을 활용해 실시간으로 시스템을 제어하는 **'피드백 루프'**를 구현하고, 그 성능 한계를 알아보는 것이 목표였습니다.

궁극적으로는 **아주 빠르게 반응해야 하는 시스템(Hard Real-Time System)**을 설계할 때, **화면 분석 속도**와 **입력 지연**을 어떻게 최소화하고 제어 안정성을 확보할 수 있는지 실제 코드로 검증하고자 했습니다.

주요 기술 학습 목표

기술 요소 핵심 내용
핵심 라이브러리 Python (OpenCV, mss, PyQt5)
제어 이론 P-제어, 가중 이동 평균 (WMA) 필터링
보안적 관점 탐지 회피 기법 연구 및 안티 치트 시스템 분석

2. 시스템 아키텍처 및 구조 (어떻게 만들고 싶었나?)

우리는 화면 캡처, 분석, 제어 명령의 세 단계를 **서로 간섭하지 않게** 분리하여, **작업을 동시에 처리(병렬 처리)**하는 구조를 목표로 했습니다. 이렇게 해야 한쪽이 느려져도 전체 시스템 속도에 영향을 덜 주기 때문입니다.

핵심 구성 요소 상세 설명

  • **데이터 수집/제어 스레드 (Detector Thread):** **가장 빠르고** 핵심적인 역할을 합니다. 화면을 분석하고 즉시 마우스 제어 명령을 운영체제(OS)에 전달합니다. **핵심 이유:** 화면 캡처와 제어는 1ms의 지연도 치명적이기 때문에, GUI 같은 무거운 작업과는 완전히 분리하여 전담시켰습니다.
  • **사용자 인터페이스 (GUI - PyQt5):** 시스템 상태를 보여주고, 사용자가 **'스무딩 값'** 같은 주요 설정값을 실시간으로 바꿀 수 있게 해줍니다. **핵심 이유:** 제어 스레드의 속도를 떨어뜨리지 않으면서도, 시스템을 **모니터링**하고 **디버깅**하기 위한 창구가 필요했습니다.
  • **스레드 간 통신 (PyQt5 Signal/Slot):** 서로 다른 두 개의 스레드가 안전하게 데이터를 주고받는 방식입니다. **핵심 이유:** 멀티스레딩 환경에서 데이터를 동시에 건드리면 프로그램이 멈추거나 오작동(Race Condition)할 수 있어, PyQt5의 안전한 통신 방식을 사용했습니다.

3. 구현 방식 및 동작 원리 (어떻게 동작하고 확인했나?)

시스템은 화면을 **100번 이상** 캡처하고, 타겟과의 오차를 계산한 다음, **패턴 없는 움직임**으로 제어 명령을 실행하는 방식으로 동작합니다.

고성능 ROI 기반 CV 파이프라인 (속도 향상의 비밀)

  • **ROI (관심 영역) 캡처:** 전체 화면이 아닌, **꼭 봐야 하는 부분(화면 중앙의 작은 영역)**만 mss로 빠르게 캡처합니다. **심화:** 일반적인 스크린샷은 수백 ms가 걸리지만, mss는 운영체제의 고속 API를 직접 사용해 **캡처 지연을 1~2ms로** 낮춥니다.
  • **HSV 색 공간 처리:** CV 분석 시, 주변 환경의 밝기 변화에 영향을 덜 받도록 RGB 대신 **HSV 색 공간**에서 타겟 색상을 검출합니다.

P-제어와 WMA 스무딩 (왜 부드럽고 정확한가?)

  • **P-제어 적용:** 목표물과의 거리(오차)에 비례해서 제어 명령(마우스 이동)을 내리는 **P-제어(Proportional Control)**를 기본 로직으로 사용합니다. **심화:** 목표물에 가까워질수록 느려지게 하여 **오버슈트(OverShoot, 목표물을 지나치는 현상)**를 줄이고 제어 안정성을 높였습니다.
  • **WMA(가중 이동 평균) 스무딩:** 단순한 '느리게 움직이기'가 아니라, **최근의 움직임에 더 높은 가중치**를 주는 필터링(WMA)을 적용하여 제어 움직임을 자연스럽고 안정적이게 만듭니다.

본 프로젝트의 주요 우수 기능 (V2.0)

랜덤 가변 스무딩 기법 (탐지 회피 연구)

**smoothing_factor** 값을 주기적으로 **무작위**로 변경하여 제어 궤적에 **규칙적인 패턴**이 생기는 것을 막았습니다. **연구 의의:** 이는 기계적인 움직임을 탐지하는 **방어 시스템(Anti-Cheat)**의 분석 기준을 우회하기 위한 학술적 연구 목적으로, 제어 시스템의 비선형성을 높이는 방법을 탐구한 것입니다.

4. 목표 근접도 및 성능 분석 (얼마나 잘 만들었나?)

V2.0은 **100 FPS**에 근접한 속도와 **10ms** 수준의 낮은 지연 시간을 달성했습니다. 이는 **사람의 수동 조작 평균 명중률(40%)**을 상회하는 **55%**의 CV 정확도를 기록하며 실시간 제어 목표에 성공적으로 도달했습니다.

시스템 요소 / 항목 사람 (수동 조작 평균) 버전 V1.0 (초기) 버전 V2.0 (현재) V3.0 목표 (연구)
처리 속도 (FPS) $\sim 60 \text{FPS}$ $\sim 45 \text{FPS}$ $\mathbf{60 \sim 100 \text{FPS}}$ $> 144 \text{FPS}$
End-to-End Latency $\sim 50 \text{ms}$ $\sim 20 \text{ms}$ $\sim 10 \text{ms}$ $< 5 \text{ms}$
명중률 (CV 정확도) $\sim 40\%$ (평균) $\sim 38\%$ $\mathbf{\sim 55\%}$ $\sim 70\%$ 이상 (YOLOv8 기반 목표)

**주목할 점:** V2.0은 사람의 **평균 반응 속도($\sim 50 \text{ms}$)**보다 훨씬 빠른 **$\sim 10 \text{ms}$의 Latency**를 기록했습니다. 이는 **제어 시스템의 고속 처리 능력**을 명확히 보여줍니다.

5. 결론 및 향후 연구 방향

주요 성과 및 얻은 교훈 (1-2줄 요약)

  • **주요 성과:** 고속 ROI 캡처와 P-제어/WMA 필터를 결합하여 **저지연, 고정확도의 실시간 제어 시스템**을 구축하고 성능을 검증했습니다.
  • **얻은 교훈:** 알고리즘의 복잡성보다 **데이터 파이프라인의 효율성(I/O 최적화)**이 실시간 제어 성능의 가장 큰 핵심이라는 것을 깨달았습니다.

향후 심화 연구 방향

  • **PID 제어 루프 확장:** 잔류 오차를 제거하고 제어 응답을 더 정밀하게 다루는 **PID 제어**로 확장하여 제어 이론을 심층 연구할 예정입니다.
  • **딥러닝 기반 객체 검출:** GPU 가속 AI 모델(예: YOLOv8)을 통합하여 CV 정확도를 획기적으로 향상시키고, **하드웨어 가속**을 통한 Latency 최소화를 목표로 합니다.

6. 연구 윤리 선언

기술적 접근 방식 (White-Box)

본 시스템은 외부 메모리 접근 없이, 오직 **화면 출력**을 분석하는 **순수한 컴퓨터 비전(CV) 방식**입니다. 이는 분석의 투명성을 극대화합니다.
